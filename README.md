# Cost Calculator

Учебный проект по кейсу №2 **«Расчёт себестоимости при позаказном производстве»**, выполненный в рамках дисциплины **«Основы проектной деятельности»** в РТУ МИРЭА.

Проект представляет собой MVP веб-сервиса для загрузки производственных данных из CSV-файлов, проверки их корректности, расчёта себестоимости заказа и формирования отчёта с детализацией по статьям затрат.

## Описание проекта

`Cost Calculator` предназначен для расчёта фактической себестоимости производственного заказа на основе трёх групп затрат:

- материалы;
- трудозатраты;
- накладные расходы.

Система позволяет загружать CSV-файлы с данными по заказу, валидировать их, сохранять корректные данные, фиксировать ошибки импорта и рассчитывать итоговую себестоимость.

Расчёт выполняется по формуле:

```text
Себестоимость = Материалы + Труд + Накладные
```

Также рассчитывается маржа заказа:

```text
Маржа = Цена заказа - Себестоимость
```

## Цель проекта

Повысить точность, прозрачность и скорость расчёта фактической себестоимости заказов в позаказном производстве за счёт автоматизированной обработки данных о материалах, трудозатратах и накладных расходах.

## CSV-файлы

Проект работает с тремя типами CSV-файлов.

### BOM

Файл состава заказа содержит материалы, необходимые для выполнения производственного заказа.

Пример структуры:

```csv
Order_id;Quantity;Unit_cost;Material_Code
order-101;150.5;12.50;STEEL-SHEET-2MM
order-101;8.0;350.00;PAINT-RAL9010
order-101;500;2.30;ELECTRODE-3MM
```

Поля:

- `Order_id` — идентификатор заказа;
- `Quantity` — количество материала;
- `Unit_cost` — стоимость единицы материала;
- `Material_Code` — код материала.

### Labor

Файл трудозатрат содержит данные о часах работы и ставках.

Пример структуры:

```csv
Order_id;Rate;Hours
order-101;450.00;8
order-101;650.00;4
```

Поля:

- `Order_id` — идентификатор заказа;
- `Rate` — часовая ставка;
- `Hours` — количество отработанных часов.

### Overhead

Файл накладных расходов содержит косвенные затраты по заказу.

Пример структуры:

```csv
Order_id;Date;Type;Amount
order-101;2026-03-05;Доставка спецоборудования;4500.00
order-101;2026-03-12;Консультация технолога;3000.00
```

Поля:

- `Order_id` — идентификатор заказа;
- `Date` — дата расхода;
- `Type` — тип накладного расхода;
- `Amount` — сумма расхода.

## Валидация данных

При импорте CSV-файлов выполняются проверки:

- наличие обязательных столбцов;
- отсутствие пустых обязательных значений;
- корректность числовых полей;
- положительные значения для количества, часов, ставок и сумм;
- корректность формата даты;
- соответствие строк существующему заказу;
- корректный тип загружаемого файла.

Если во время импорта обнаружены ошибки, они фиксируются в журнале. При этом последняя корректная версия данных сохраняется.

## Методы распределения накладных расходов

В проекте предусмотрены два метода распределения накладных расходов.

### Пропорционально трудоёмкости

```text
Накладные по заказу = (Все накладные расходы за период / Всё время работы) * Время производства конкретного заказа
```

### Пропорционально прямым затратам

```text
Накладные по заказу = (Все накладные расходы за период / Суммарная стоимость материалов) * Стоимость материалов по заказу
```

Третий метод распределения накладных расходов по фиксированным ставкам подразделений в рамках MVP не реализуется.

## Статусы заказа

В проекте используется статусная модель заказа:

```text
opened -> in_progress -> completed -> closed
```

Для заказов со статусами `completed` и `closed` ограничивается редактирование данных и импорт новых CSV-файлов.

## Отчёт по себестоимости

Система формирует детализированный отчёт **«Себестоимость заказа»**.

Отчёт содержит:

- материалы;
- трудозатраты;
- накладные расходы;
- итоговую себестоимость;
- цену заказа;
- маржу;
- выбранный метод распределения накладных расходов.

## KPI проекта

Для оценки результата проекта были определены KPI.

### Точность калькуляции

```text
|Фактическая себестоимость - Рассчитанная себестоимость| / Фактическая себестоимость
```

Целевое значение: не более 5%.

### Скорость расчёта

```text
Время формирования отчёта «Себестоимость заказа»
```

Целевое значение: не более 5 секунд.

### Полнота учёта затрат

```text
Количество заказов с полностью заполненными полями / Общее количество заказов * 100%
```

Целевое значение: 100%.

## Запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/jabrail059/cost-calculator.git
cd cost-calculator
```

### 2. Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Пример переменных окружения:

```env
SERVER_ADDR=:8080
FRONTEND_API_ADDR=:3000
APP_PORT=8080
API_PORT=3000

DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=change-me
DB_NAME=proddb
DB_SSLMODE=disable

JWT_SECRET=change-me

ONEC_URL=http://onec-server:8080/generate
ONEC_TIMEOUT=30s

UPLOAD_MAX_MEMORY=10485760
WEB_DIR=./web
```

Перед запуском рекомендуется заменить значения `DB_PASSWORD` и `JWT_SECRET`.

### 3. Запуск через Docker Compose

```bash
docker compose up --build
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8080
```

API compatibility server используется на порту:

```text
http://localhost:3000
```

## Модель данных

В рамках MVP используются основные сущности:

- `orders` — производственные заказы;
- `bom` — состав заказа и материалы;
